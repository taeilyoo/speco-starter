# Security Rules

모든 작업에서 반드시 준수해야 하는 보안 규칙입니다.

## 필수 규칙

### 1. 비밀 정보 금지
- API 키, 비밀번호, 토큰을 코드에 직접 작성 금지
- 모든 비밀은 환경 변수로 관리
- `.env` 파일은 절대 커밋하지 않음

```typescript
// ❌ Bad
const API_KEY = 'sk_live_abc123';

// ✅ Good
const API_KEY = process.env.API_KEY;
```

### 2. 입력값 검증
- 모든 사용자 입력은 서버에서 재검증
- SQL 쿼리는 Parameterized Query 사용
- 화이트리스트 방식 검증 선호

### 3. 인증/인가
- 모든 API 엔드포인트에 인증 검사
- 권한 검사 누락 확인
- JWT 토큰 만료 시간 설정

### 4. 의존성 관리
- 새 패키지 설치 전 `npm audit` 실행
- 알려진 취약점이 있는 버전 사용 금지

## 코드 리뷰 시 확인 사항

- [ ] 하드코딩된 비밀 없음
- [ ] 모든 입력값 검증됨
- [ ] 인증/인가 로직 존재
- [ ] 에러 메시지에 내부 정보 노출 없음
- [ ] 로그에 민감 정보 없음
