# 리팩토링 원칙 (Refactoring Principles)

## 핵심 규칙

### 파일 크기 제한
| 항목 | 제한 |
|------|------|
| 파일 | 300줄 초과 금지 |
| 함수 | 50줄 초과 금지 |
| 클래스/컴포넌트 | 200줄 초과 금지 |

### 단일 책임 원칙 (SRP)
- 하나의 파일/함수는 하나의 책임만
- 여러 이유로 변경될 수 있다면 분리 필요

### 죽은 코드 제거
- 사용되지 않는 함수/변수 즉시 제거
- 주석 처리된 코드 삭제
- 불필요한 import 제거

## 리팩토링 우선순위

### 1순위: 긴급
- [ ] 보안 취약점
- [ ] 성능 병목
- [ ] 중복 코드 (3회 이상)

### 2순위: 높음
- [ ] 300줄 초과 파일
- [ ] 50줄 초과 함수
- [ ] 깊은 중첩 (3단계+)

### 3순위: 중간
- [ ] 네이밍 개선
- [ ] 타입 안전성 강화
- [ ] 테스트 추가

## 리팩토링 기법

### 함수 추출 (Extract Function)
긴 함수를 작은 단위로 분리

### Early Return (가드 절)
깊은 중첩 대신 조건 불충족 시 즉시 반환

### 조건문을 다형성으로
반복되는 switch 문을 전략 패턴으로 변환

## 안전 수칙

- [ ] 변경 전 테스트 통과 확인
- [ ] 작은 단위로 점진적 변경
- [ ] 변경마다 테스트 실행
