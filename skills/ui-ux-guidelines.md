# UI/UX 디자인 가이드라인

## 디자인 원칙

1. **일관성**: 동일 기능은 동일 패턴으로
2. **피드백**: 모든 인터랙션에 즉각적 반응
3. **예측 가능성**: 사용자가 결과를 예상 가능
4. **효율성**: 최소 클릭으로 목표 달성
5. **접근성**: 모든 사용자가 이용 가능

## 컴포넌트 규칙

### 버튼 계층
| 유형 | 용도 | 스타일 |
|------|------|--------|
| Primary | 주요 액션 | 배경색 채움 |
| Secondary | 보조 액션 | 테두리만 |
| Tertiary | 덜 중요한 액션 | 텍스트만 |

### 버튼 상태
- Default / Hover / Active / Disabled / Loading

### 입력 필드 상태
- Default / Focus / Filled / Error / Success / Disabled

## 간격 시스템 (4px 기반)
```
4px  - 아이콘 내부
8px  - 관련 요소 간격
16px - 컴포넌트 내부 패딩
24px - 섹션 내 요소 간격
48px - 섹션 간 간격
```

## 반응형 브레이크포인트
| 디바이스 | 범위 |
|----------|------|
| Mobile | 0-767px |
| Tablet | 768-1023px |
| Desktop | 1024px+ |

## 접근성 (WCAG 2.1 AA)
- 색상 대비: 본문 4.5:1, 대형 텍스트 3:1
- 터치 타겟: 최소 44x44px
- 키보드: 모든 요소 Tab 접근 가능
- 스크린리더: alt, aria-label 필수

## 인터랙션 패턴

### 로딩 상태
| 대기 시간 | 권장 패턴 |
|-----------|-----------|
| 0-300ms | 즉시 표시 |
| 300ms-1s | 버튼 내 스피너 |
| 1-3초 | 스켈레톤 UI |
| 3초+ | 진행률 표시 |

### 에러 처리
- 인라인 에러: 필드 바로 아래
- 토스트: 일시적 에러
- 항상 해결 방법 제시

### 빈 상태 (Empty State)
- 설명적 일러스트/아이콘
- 현재 상태 설명
- 행동 유도 CTA 버튼
