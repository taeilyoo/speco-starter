# 서비스 기획 원칙

## 기획 문서 구조

1. **문제 정의**: 해결하려는 사용자 문제
2. **목표 설정**: KPI 및 성공 측정 기준
3. **사용자 정의**: 페르소나, 사용자 여정
4. **기능 명세**: 우선순위 포함 기능 목록
5. **화면 설계**: 와이어프레임, 플로우

## 우선순위 프레임워크

### RICE 스코어
`Score = (Reach × Impact × Confidence) / Effort`

| 항목 | 설명 |
|------|------|
| Reach | 영향받는 사용자 수 |
| Impact | 0.25 / 0.5 / 1 / 2 / 3 |
| Confidence | 100% / 80% / 50% |
| Effort | 인주(person-weeks) |

### MoSCoW 방법론
| 분류 | 의미 |
|------|------|
| Must | 필수, 없으면 출시 불가 |
| Should | 중요, 대안 존재 |
| Could | 있으면 좋음 |
| Won't | 이번 범위 제외 |

## 사용자 플로우 규칙

- 시작점/종료점 명확히
- **3클릭 규칙**: 핵심 기능까지 3클릭 이내
- 분기점마다 조건 명시
- 에러 케이스 별도 표기

## 기능 명세 템플릿

```markdown
## 기능명: [이름]

### 사용자 스토리
As a [사용자 유형]
I want [기능/행동]
So that [달성하려는 가치]

### 수용 조건
Scenario: [시나리오명]
  Given [사전 조건]
  When [사용자 행동]
  Then [예상 결과]
```

## 페르소나 템플릿
- 기본 정보 (나이, 직업, 기술 수준)
- 목표 (Goals)
- 불편점 (Pain Points)
- 행동 특성
